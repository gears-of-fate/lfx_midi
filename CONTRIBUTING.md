# Contributing Guidelines

Thank you for your interest in contributing to this Toymacker-System library! Any kind of help (bug reports, new documentation, code fixes) is highly appreciated.

## How to Report a Bug
* Use the "Issues" section of this repository.
* Before opening a new issue, please check if the bug has already been reported by someone else.
* Describe the problem clearly.
* Include the exact steps to reproduce the error.
* Specify your operating system and the version of your development environment.

## How to Propose a Change (Pull Request)
* Fork the repository and create a specific branch for your changes (e.g., `feature/new-feature` or `bugfix/fix-error`).
* Ensure the code is clean, readable, and well-commented.
* If you add a new source file, make sure to include the Toymacker-System GPLv3 license header at the top of the file.
* Write a clear and concise commit message.
* Open the Pull Request explaining in detail the problem you solved or the feature you added.

## Code Style
* Maintain consistency with the formatting style already present in the project.
* If possible, add unit tests for the new features you are proposing.

## Code of Conduct
We encourage you to maintain a respectful and collaborative tone in all discussions. Our goal is to learn together and grow the project in a welcoming environment.

## Contact
For security issues or direct contact not related to public bugs, you can reach out to: dev@toymacker-system.dev
